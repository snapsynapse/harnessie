Key invariants:
- Guarantees live in code (registry, loop, OS layer), never in prompts.
- Routing is config, not model self-assessment; missing tiers fail early.
- Task packets are offers: side effects lock until accept_task; declining is legitimate and never escalates the route.
- Agents own the files they create, never another agent's; operator lanes are locked; the ownership ledger is outside the workspace jail.
- Contested decisions halt for human arbitration; no agent and no code path authors an Arbitration section; dissent is never deleted.
- Verifier verdict and stance parsing fail closed; resume skips only verified successes; workers never self-certify gated completion.
- Shell and deterministic checks require a usable OS sandbox (fail closed without one).
- events.jsonl is hash-chained; memory facts carry stamped provenance and verify_by expiry; expiry is approval-gated and archival-only.
Working rules:
- Use `rg` for search.
- Keep changes scoped to the harness invariant being improved.
- Update docs and tests in the same change when behavior changes.
- Add eval scenarios for new guarantees, especially risky and recovery behavior; a governance mechanic without a red-then-green scenario pair does not merge.
- Do not stage `.agents/`, `.codex/`, `runs/`, `workspace/`, `.pytest_cache/`, or local scratch unless explicitly requested.
